Simple Optical Character Recognition

This notebook uses PyTorch tensors to process images from the MNIST dataset. This dataset contains 28x28 black and white images of handwritten numerical digits. The objective of this notebook is to see if creating a manual application (without using any artificial neural network) can sufficiently work as a digit classifier for all digits from 0 to 9.

Defining a measure of similarity

Now that average images for each digit can be seen, a similarity measure to assess which digit is most similar to the input image has to be defined. In other words, a loss function needs to be defined where the actual digit of the input image gets assigned the least loss, compared to the remaining digits.

Here, similarity is measured by value difference between each pixel of the input image and the ideal image of a digit, on average. The ideal image with the least average value difference i.e. least loss will be the prediction. To preserve the difference as distance, squared difference should be considered (Alternatively, absolute value can also be considered).

Defining a prediction function

Using the above illustration, a prediction can be made by calculating the average distance of an input image with average images of all 10 handwritten digits. Prediction is therefore the average image with the least loss.

In[31]:
# pass the input image as a path
def predict(image):
    image_tensor = tensor(Image.open(image))
    
    # Calculation for average squared difference 
    loss_0 = (pow(image_tensor.float() - avg_0.float(),2)).mean((-1,-2))
    loss_1 = (pow(image_tensor.float() - avg_1.float(),2)).mean((-1,-2))
    loss_2 = (pow(image_tensor.float() - avg_2.float(),2)).mean((-1,-2))
    loss_3 = (pow(image_tensor.float() - avg_3.float(),2)).mean((-1,-2))
    loss_4 = (pow(image_tensor.float() - avg_4.float(),2)).mean((-1,-2))
    loss_5 = (pow(image_tensor.float() - avg_5.float(),2)).mean((-1,-2))
    loss_6 = (pow(image_tensor.float() - avg_6.float(),2)).mean((-1,-2))
    loss_7 = (pow(image_tensor.float() - avg_7.float(),2)).mean((-1,-2))
    loss_8 = (pow(image_tensor.float() - avg_8.float(),2)).mean((-1,-2))
    loss_9 = (pow(image_tensor.float() - avg_9.float(),2)).mean((-1,-2))
    
    # Storing deviations as losses
    losses = [loss_0, loss_1, loss_2, loss_3, loss_4, loss_5, loss_6, loss_7, loss_8, loss_9]
    
    # Returning digit with the least loss
    return losses.index(min(losses))

# Defining accuracy function
def calc_acc(digit):
    digit = str(digit)
    digit_path = (path/'testing'/digit).ls()
    predictions = [predict(i) for i in digit_path]
    
    # number of correct predictions
    correct_pred = [j==float(digit) for j in predictions]
    
    # accuracy
    return sum(correct_pred)/len(predictions)

Conclusion

Without using any artificial neural networks, a simple digit classifier using handwritten images is created in this notebook. This was achieved through image processing techniques using PyTorch tensors. It is seen that the accuracy for this approach is just around 81%. While the performance is not bad, the problem with this approach is that it's not learning from its mistakes. But this is exactly the advantage of using artificial neural networks(ANNs). So it would be interesting to see how ANNs perform compared to this approach.
